VIPER

Visual Intercept & Precision Engagement Response

1. Project Overview

VIPER is a computer vision-based simulated anti-aircraft defense system. The application utilizes a live webcam feed and real-time object detection (YOLOv8) to identify incoming 'airplane' targets. Once a target is detected, it renders a tracking HUD, engages a synthetic radar system, and interacts with system audio to sound alarms and simulated weapon fire.

2. System Architecture

The system is divided into a Python Flask backend serving the ML inference and an HTML/JS frontend rendering the operator UI.

  • Backend (Python/Flask): Handles standard HTTP routing, streams a Motion JPEG (mjpeg) live webcam feed, runs YOLOv8 model inference for target identification, and manages local system sound execution using Windows ctypes.windll.winmm to bypass browser playback restrictions.
  • Frontend (HTML/CSS/JS): Displays a tactical dashboard. Pings the backend via AJAX to retrieve near real-time target telemetry points. The DOM updates dynamically to show reticles, radar blips, confidence levels, and weapon readiness.

3. Core Logic & Engagement Rules

The system evaluates threats based on the confidence level of the YOLOv8 detection for the "airplane" class (Class 4 in the COCO dataset).

Confidence Level Target State System Response
Conf < 30% None / Scanning Turret Idle, Radar clear, No targets displayed.
30% ≤ Conf < 50% Target Acquired (Tracking) Warning Banner Displays.
Target Reticle & Radar Blip activated.
Threat: Low/High. Weapon tag: TRACKING/READY.
Conf ≥ 50% Engaged (Fire Mode) Target Locked UI active.
Weapon status changed to "FIRING" (Red visuals).
System sound: Looping Machine Gun Fire.

Note: A siren sound effect runs exactly once during the operational lifecycle of the app when the first detection event occurs.

4. File & Directory Structure

  • app.py: The core application runner. Initializes YOLO model, handles threads for webcam capturing and inferences, API Endpoints, and runs Python `ctypes` for sound triggering.
  • yolov8n.pt: The Ultralytics Nano model weights used for detection. It's lightweight enough to allow real-time classification locally on CPU/GPU.
  • templates/index.html: The operator Dashboard. A single-page tactical UI encompassing the video feed, telemetry parameters, radar visualization, and weapon systems indicators.
  • static/: Directory holding static graphical and audio assets:
    • gun_active.png: Firing weapon asset.
    • gun_inactive.png: Idle weapon asset.
    • sound/siren.mp3: Alarm sound executed upon initial detection.
    • sound/mechine_gun.mp3: Attack weapon sound looped during high-confidence threat state.

5. Key API Endpoints

  • GET /: Dashboard entry point. Serves index.html.
  • GET /video_feed: Streams the webcam feed processed via OpenCV. Bounding boxes and confidence overlays are drawn server-side here. Yields a multipart/x-mixed-replace stream.
  • GET /get_coords: Provides current global state telemetry logic (in JSON). Example response: {"detected": true, "x": 320, "y": 240, "conf": 65.5}. The frontend pings this continually to update the UI variables.

6. Technical Prerequisites

Ensure the following Python libraries are installed to run the application:

  • flask: Web server framework
  • opencv-python (cv2): Real-time imagery and rendering
  • ultralytics: Enables YOLOv8 logic

OS requirement: Because of `ctypes.windll.winmm` usage in backend sound controllers, this app requires a Windows environment to process sound outputs.
